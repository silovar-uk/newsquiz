# News Context Quiz

ニュースを「見た」で終わらせず、背景まで説明できる知識に変えるスマホ優先の四択クイズアプリです。

- 30問を基本に、JSONを貼り付けて問題セットを取り込む
- 選択直後は「まずここだけ」で結論を表示。選択肢の理由・背景・重要ワードは必要なときだけ深掘りできる
- 国内外ニュースだけでなく、スポーツ、芸能・カルチャー、地域、地理、経済、制度、科学まで扱う
- テーマ指定時は「主題中心＋少しだけ背景」に自動調整
- 「3本本当＋1本うそ」のまとめ読み問題に対応
- 回答時間、ヒント使用、復習登録、カテゴリー別の苦手を保存。不正解・ヒント使用は自動で復習候補へ
- IndexedDBにローカル保存。問題セットと履歴をJSONでバックアップ可能
- GitHub PagesとPWA対応
- ホームでは「今日の次の一手」を表示。途中再開、最新セット、5問だけ復習を優先表示

## 使い方

1. ホームの **今日の30問を準備する** から、作成画面を開く。
2. 「おまかせバランス」または「テーマ深掘り」と問題数を選ぶ。細かい配分は **バランスを調整する** を開いたときだけ変更できる。
3. **プロンプトをコピー** し、ChatGPTなどに貼る。
4. AIから返ってきたJSONだけをコピーし、アプリの **取込** 画面に貼る。
5. エラー・警告を確認して取り込む。
6. ホームの **今日の次の一手** からクイズを開始する。
7. 不正解・ヒント使用は自動的に復習候補へ入り、ホームや履歴から5問だけ解き直せる。


最初から、動作確認用の「ニュースを読むための基礎30問」が入っています。

## 出題の考え方

### おまかせバランス（30問の初期値）

- ニュースを直接問う問題：21問
  - 国内ニュース：4問以上
  - 海外ニュース：4問以上
  - スポーツ：4問以上（国内外を混ぜる）
  - 芸能・カルチャー：4問以上（国内外を混ぜる）
  - 日本・世界の地域基礎：合計6問以上
  - 政治・経済・科学文化：合計6問以上
- つながる知識：6問
  - 制度、地理、比較、歴史、仕組み
- まとめ読み：3問
  - 「次のうち、事実と異なるものはどれ？」
  - 4択のうち、3つは事実、1つは誤り

### テーマ深掘り（30問の初期値）

- テーマを直接問う問題：24問
- 背景・比較・周辺知識：4問
- まとめ読み：2問

テーマを渡したのに、関係の薄い知識クイズばかりになるのを避ける設計です。


## UI/UXの更新（v1.1）

- ホームの最上部を「今日の次の一手」に変更。問題セットがない日は準備、途中なら再開、セットがあれば解答開始を最優先に表示。
- 解説を二段階化。解答直後は結論だけ表示し、選択肢の理由・背景・重要ワード・出典は「詳しく見る」から開く。
- 「3本本当＋1本うそ」は専用表示。解答後に4本すべての真偽と解説を一度に確認できる。
- 解答後は画面下の固定ボタンで次へ進めるため、長い解説を読んだ日でも戻る操作が不要。
- 不正解・ヒント使用・手動保存から、ホームと履歴に「5問だけ復習する」を自動表示。

## JSONの要点

1セットの最小構造は以下です。実際の完全なサンプルは `public/sample-quiz.json` を参照してください。

```json
{
  "schemaVersion": 1,
  "id": "20260704-example",
  "title": "今日のニュース30問",
  "mode": "balanced",
  "createdAt": "2026-07-04T00:00:00.000Z",
  "validAsOf": "2026-07-04",
  "questionBalance": { "direct": 21, "context": 6, "roundup": 3 },
  "questions": [
    {
      "id": "q01",
      "type": "standard",
      "category": "world-news",
      "learningFocus": "direct",
      "difficulty": 1,
      "prompt": "問題文",
      "hint": "答えを直接言わない短いヒント",
      "choices": [
        { "id": "a", "text": "選択肢A", "explanation": "Aの解説" },
        { "id": "b", "text": "選択肢B", "explanation": "Bの解説" },
        { "id": "c", "text": "選択肢C", "explanation": "Cの解説" },
        { "id": "d", "text": "選択肢D", "explanation": "Dの解説" }
      ],
      "correctChoiceId": "a",
      "shortExplanation": "最初に読む結論",
      "background": "ニュースとつながる背景知識",
      "keywords": [
        { "term": "重要ワード", "shortDefinition": "一言でいうと", "searchQuery": "検索語句" }
      ],
      "sources": [
        { "name": "出典名", "title": "資料タイトル", "url": "https://example.com/", "publishedAt": "2026-07-04" }
      ]
    }
  ]
}
```

`type` は以下の2種類です。

- `standard`：通常の四択
- `false_news`：3つの事実と1つの誤りを見抜く四択

`learningFocus` は以下の3種類です。

- `direct`：テーマやニュースを直接問う
- `context`：背景・比較・地理・制度を問う
- `roundup`：まとめ読み問題

## GitHub Pagesへ公開

このリポジトリには `.github/workflows/deploy-pages.yml` を同梱しています。

1. GitHubで新規リポジトリを作成する。
2. このフォルダの中身をpushする。
3. GitHubの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** にする。
4. `main` ブランチへのpush後、Actionsの `Deploy to GitHub Pages` が成功すれば公開される。

アプリは `base: './'` にしているため、`https://ユーザー名.github.io/リポジトリ名/` のようなプロジェクトページでもアセットパスが崩れにくい構成です。

## データの扱い

- 問題セット、解答履歴、プロンプト設定はブラウザのIndexedDBへ保存する。
- サーバーや外部データベースは使わない。
- 端末変更やブラウザデータ削除に備え、**履歴 → バックアップを書き出す** でJSON保存を推奨。
- ニュース問題の正確性は、生成時の出典・情報基準日で担保する設計。アプリは出典の実在性そのものを自動検証しない。
